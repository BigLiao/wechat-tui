# CHANGELOG.md

本文件记录 `wechat-tui` 的用户可见变更和重要内部变更。

格式约定：

- 顶部保留 `## [Unreleased]`，记录尚未发布的变更。
- 发布时将 `Unreleased` 内容移动到 `## [x.y.z] - YYYY-MM-DD`。
- 每个版本按需使用 `Added`、`Changed`、`Fixed`、`Docs`、`Internal` 分类。
- 每条变更使用简短中文描述，必要时补充对应提交或模块名。
- 空分类不保留。

## [Unreleased]

## [0.5.0] - 2026-06-15

### Added

- 最近会话列表命令面板新增 `/readall`，可一键将当前账号下所有会话标记为已读。

### Changed

- 启动流程改为立即显示启动界面，将本地 session 读取、协议初始化和是否需要二维码登录的等待过程隐藏在启动界面内。
- 保存登录态恢复时继续展示启动界面，并保持 3 秒最短停留时间；需要重新登录时再切换到二维码登录界面。

### Fixed

- 修复没有 `protocol_id` 的遗留会话未读数无法随当前联系人会话一起清除，导致已进入会话后仍显示未读的问题。
- 修复聊天页未读摘要会显示 0 未读的 Tab 返回目标，造成普通会话看起来像未读提醒的问题。

### Docs

- README 和 CLI 帮助补充 `/readall` 命令说明。

### Internal

- 增加批量已读、启动页时序、遗留会话未读合并和命令面板渲染测试。

## [0.4.1] - 2026-06-12

### Added

- 新增 Pull Request CI，在 PR 中自动执行类型检查、测试和构建。

### Changed

- 将 SQLite 持久化从 Node.js 内置同步 `node:sqlite` 切换为 `sqlite`/`sqlite3` 异步驱动，减少联系人和消息写入对 TUI 渲染与输入的阻塞。
- 协议登录、联系人、消息和退出事件统一串行处理，并合并消息洪峰中的渲染刷新，降低异步事件交错导致的状态抖动。

### Fixed

- 修复退出登录事件可能抢在已排队登录或消息处理前执行，导致界面重新显示为已连接状态或持久化过程使用错误账号状态的问题。
- 修复协议消息连续到达时仍可能触发多余渲染的问题。

### Docs

- 更新 README 和 AGENTS 中的 Node.js 运行要求说明，移除对内置 `node:sqlite` 的描述。

### Internal

- 将 store、runtime 和相关测试改为 Promise-based 持久化调用，并增加异步写队列、协议事件串行化和文件路径解析覆盖。

## [0.4.0] - 2026-06-11

### Added

- 已保存登录态恢复成功后显示模拟微信启动图的 ASCII 启动画面；无登录态或登录态失效时优先展示二维码。

### Fixed

- 跳过重复到达的协议消息，避免重复刷新、渲染和媒体下载造成操作卡顿。
- 合并后台消息和媒体下载触发的渲染，减少消息洪峰时的终端重绘。

### Internal

- 优化消息列表查询和 SQLite 索引，降低会话消息读取成本。
- 增加启动页、重复消息跳过和渲染合并测试。

## [0.3.1] - 2026-06-03

### Fixed

- 修复群聊会话已补全真实群名后，后续稀疏消息会把标题覆盖回 `@@...` 协议 ID 的问题。

## [0.3.0] - 2026-06-03

### Added

- 聊天界面支持使用 Tab 在未读会话之间快速切换，并可用 Enter 确认、Esc 取消。
- 终端标签标题默认显示 `WeChat`，有未读消息时追加未读数。

### Changed

- 将群成员资料与联系人资料拆分建模，群成员不再被当作可直接对话的联系人出现在联系人列表中。
- 群聊消息先进入渲染流程，群资料和群成员昵称改为后台补全，减少联系人接口超时导致的新消息延迟。

### Fixed

- 修复 Web 微信同步长轮询超时时被当作致命错误，导致界面提示“同步失败”并停止接收新消息的问题。
- 修复慢速 `webwxbatchgetcontact` 请求阻塞消息分发，导致新消息长时间后才显示的问题。
- 修复群成员昵称异步回填后，历史群消息发送人无法按群维度对齐的问题。

### Internal

- 增加群成员存储、联系人隔离、Tab 未读切换、终端标题和非阻塞消息分发测试。

## [0.2.9] - 2026-06-02

### Changed

- 统一聊天正文和会话列表的消息时间展示：今天只显示 `HH:mm`，昨天显示 `昨天 HH:mm`，同年更早消息显示 `M月D日 HH:mm`，跨年消息显示 `YYYY年M月D日 HH:mm`。
- 媒体消息占位符使用更克制的主题色，降低 `[image #xxxx]`、`[file #xxxx]` 等标签的视觉干扰。

### Docs

- 完善 npm package 描述和关键词，增加中文说明和搜索关键词。

### Internal

- 增加消息时间格式化和媒体占位符渲染测试，覆盖自然日边界和 ANSI 颜色输出。

## [0.2.8] - 2026-06-01

### Changed

- 会话列表标题展示宽度增加，超长标题使用省略号截断。
- 最近会话预览前增加消息时间，今天显示 `HH:mm`，更早消息显示 `M月D日 HH:mm`。

### Fixed

- 修复 Web 微信返回稀疏或截断群名时，未优先使用完整成员列表生成群聊标题的问题。

### Docs

- README 增加 npm 版本、下载量徽章。
- 新增 MIT License，并在 npm package 元数据中声明。

### Internal

- 抽取群聊名称和成员数归一化工具，统一协议层、运行时和存储层的比较逻辑。
- 清理测试数据中的真实形态联系人、协议 ID 和文件名样例，改用测试占位数据。
