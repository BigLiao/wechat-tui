2. CORE CONCEPTS & ARCHITECTURE
================================================================================

A. COMPONENT-BASED ARCHITECTURE
   - All UI elements implement the minimal Component interface
   - Simple render() method that returns array of strings (lines)
   - Optional handleInput() for keyboard handling
   - Optional invalidate() for cache clearing
   - wantsKeyRelease property for Kitty protocol key release events

B. RENDERING SYSTEM
   - Differential Rendering: Only updates changed regions
   - CSI 2026 Synchronized Output: Atomic screen updates (no flicker)
   - Debounced Rendering: Batches multiple render requests
   - Hardware Cursor Positioning: For IME (Input Method Editor) support
   - CURSOR_MARKER mechanism: Enables CJK input at correct position

C. TERMINAL ABSTRACTION
   - Terminal interface: Minimal abstraction with columns/rows
   - ProcessTerminal: Default implementation using stdin/stdout
   - Kitty Protocol Detection: Auto-detects advanced keyboard protocol
   - Raw Mode Handling: Manages terminal capabilities

D. FOCUS & KEYBOARD MANAGEMENT
   - Focusable interface: For components requiring text cursor
   - Keyboard Input Pipeline: InputListeners → focused component
   - Key Helper: Typed key identifiers with autocomplete
   - Keybindings System: Global registry of keyboard shortcuts
   - Emacs-style editing: Built into Editor/Input components

E. OVERLAY SYSTEM
   - Overlays: Components rendered on top of normal content
   - Overlay Stacking: Topmost overlay gets focus
   - Overlay Positioning: Anchor-based or percentage-based
   - Responsive Overlays: Show/hide based on terminal dimensions
   - Overlay Compositing: Single-pass line merging

4. KEYBOARD SYSTEM
================================================================================

KEY IDENTIFIERS:
  - Special keys: escape, enter, tab, space, backspace, delete, insert
  - Navigation: up, down, left, right, home, end, pageUp, pageDown
  - Function keys: f1-f12
  - Single modifiers: ctrl, alt, shift, super (cmd/win)
  - Multi-modifiers: ctrl+shift, ctrl+alt, etc.

KEY HELPER API:
  - Key.escape, Key.enter, Key.tab, Key.space, etc. (special keys)
  - Key.ctrl("c"), Key.alt("x"), Key.shift("tab") (modifiers)
  - Key.ctrlShift("p"), Key.ctrlAlt("x") (combined modifiers)
  - String literals also work: "escape", "ctrl+c", "shift+tab"

KEYBOARD PROTOCOL:
  - Legacy terminal sequences supported
  - Kitty keyboard protocol auto-detected (CSI codes)
  - More accurate key detection with Kitty protocol
  - Key release events available (wantsKeyRelease flag)

EDITOR SHORTCUTS (Emacs-style):
  Navigation:
    - Ctrl+B/F (or Left/Right): Move cursor left/right
    - Ctrl+A/E: Move to line start/end
    - Ctrl+]: Enter jump mode (jump to character)
    - Page Up/Down: Scroll
    
  Deletion:
    - Ctrl+H (or Backspace): Delete character backward
    - Ctrl+D (or Delete): Delete character forward
    - Ctrl+U/K: Delete to line start/end
    - Ctrl+W: Delete word backward
    - Meta+D: Delete word forward
    
  Clipboard:
    - Ctrl+Y: Yank (paste)
    - Meta+Y: Yank-pop (cycle through kill ring)
    
  History:
    - Up/Down: Navigate history (when at start/end)
    - Ctrl+Z: Undo

KEYBINDINGS:
  - Global registry with tui.* namespace
  - Customizable via setKeybindings()
  - Default keybindings for all components
  - Can be remapped at runtime

5. STYLING & COLORS
================================================================================

ANSI ESCAPE SEQUENCES:
  - Pi-TUI uses standard ANSI codes
  - Chalk library (v5.6.2) recommended for convenience
  - Colors: red, green, blue, yellow, cyan, magenta, white, gray
  - Styles: bold, italic, underline, strikethrough, inverse, dim
  - True color (24-bit RGB): chalk.rgb(r,g,b) or chalk.hex("#RRGGBB")
  - Background colors: chalk.bgRed, chalk.bgGreen, etc.

THEME SYSTEM:
  - EditorTheme: borderColor, selectList theme
  - SelectListTheme: selectedPrefix, selectedText, description, scrollInfo, noMatch
  - MarkdownTheme: heading, link, code, quote, bold, italic, etc.
  - ImageTheme: fallbackColor
  - All theme functions receive text, return styled text

TEXT STYLING RULES:
  - Styles reset at end of each line
  - ANSI codes don't count toward width
  - Reapply styles on each wrapped line
  - Use utilities to handle ANSI codes correctly

11. KEY FINDINGS & INSIGHTS
================================================================================

DESIGN PHILOSOPHY:
  - Minimal core interface (Component with render/handleInput/invalidate)
  - Composable components (no inheritance-heavy hierarchies)
  - Efficient rendering (differential updates only)
  - Terminal abstraction (easy to test, adapt)

STRENGTHS:
  ✅ Simple, elegant Component interface
  ✅ Differential rendering prevents flicker
  ✅ Full keyboard protocol support including IME
  ✅ Rich set of built-in components
  ✅ Flexible overlay system
  ✅ Autocomplete and history built-in (Editor)
  ✅ Image support (Kitty/iTerm2)
  ✅ Markdown rendering with syntax highlighting hooks
  ✅ Emacs-style keyboard shortcuts
  ✅ Theme system for customization

BEST FOR:
  - Interactive CLI applications
  - Chat/messaging interfaces
  - File selection dialogs
  - Status dashboards
  - Form-based UIs
  - Multi-screen applications

CONSIDERATIONS:
  - Must respect width parameter in render()
  - ANSI codes don't count toward width
  - Styles reset at end of line (reapply per line)
  - Focus state changes dynamically
  - Performance depends on caching/invalidation

PERFECT FOR WEECHAT CLI:
  - Component-based screens (LoginScreen, ChatScreen, etc.)
  - Keyboard-driven navigation
  - Message rendering with styling
  - Chat input with history
  - Contact/conversation selection (SelectList)
  - Terminal-agnostic rendering

12. QUICK START TEMPLATE
================================================================================

import { TUI, ProcessTerminal, Text, Editor, Key, matchesKey } from "@earendil-works/pi-tui";
import chalk from "chalk";

// Setup
const terminal = new ProcessTerminal();
const tui = new TUI(terminal);

// Create components
const editor = new Editor(tui, {
  borderColor: (s) => chalk.dim(s),
  selectList: {
    selectedPrefix: (s) => chalk.inverse(s),
    selectedText: (s) => chalk.inverse(s),
    description: (s) => chalk.dim(s),
    scrollInfo: (s) => chalk.dim(s),
    noMatch: (s) => chalk.dim(s)
  }
});

editor.onSubmit = (text) => {
  console.log("Submitted:", text);
};

tui.addChild(editor);
tui.setFocus(editor);

// Handle input
tui.addInputListener((data) => {
  if (matchesKey(data, Key.ctrl("c"))) {
    tui.stop();
    process.exit(0);
  }
  return undefined;
});

tui.start();
