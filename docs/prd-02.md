微信 CLI 消息工作台需求文档 2.0

1. 产品定位

本项目是一款运行在命令行中的微信消息工作台。

它不是传统聊天软件，也不是全屏 TUI，而是一个 轻量清屏重绘式 CLI 客户端：

* 默认展示最近会话列表
* 方向键选择会话
* Enter 进入聊天
* 当前屏幕只展示当前会话内容
* 其他会话的新消息只更新状态
* Esc 返回会话列表
* 支持联系人搜索并进入会话

核心目标是把微信消息处理变成一种终端里的高效工作流。

⸻

2. 核心使用场景

2.1 启动与登录

用户启动程序后：

1. 显示微信扫码登录二维码
2. 用户扫码确认登录
3. 登录成功后进入会话列表视图
4. 程序开始监听新消息

登录后默认不是消息流，而是最近会话列表。

⸻

2.2 会话列表

登录成功后展示最近几个会话。

会话列表需要展示：

* 会话名称
* 未读数量
* 最后一条消息摘要
* 群聊消息的发送人
* 当前选中的会话高亮
* 底部状态栏

示意：

WeChat TUI
Recent Chats
> 项目A群        3    王五: 新字段是 campaignId
  老板           1    下午三点有空吗？
  妈妈           0    晚上回来吃饭吗？
  文件传输助手   0    [file] design.png
online │ unread 4 │ ↑/↓ select │ Enter open │ /search │ q quit

支持操作：

↑ / ↓      选择会话
Enter      打开会话
/search    搜索联系人
q          退出

⸻

2.3 聊天窗口

进入会话后，清空屏幕，只展示该会话的消息列表。

示意：

Chat: 老板
[09:12] 老板
  今天那个活动页进度怎么样？
[09:14] You
  主视觉和适配方案已经确定了，今天可以把首屏做出来。
[14:21] 老板
  下午三点有空吗？
online │ 当前: 老板 │ unread: 项目A群(3), 妈妈(1) │ Esc chats
老板 >

聊天窗口中：

* 普通文本直接作为消息发送
* Enter 发送
* Esc 返回会话列表
* 当前会话的新消息追加展示
* 其他会话的新消息不插入正文，只更新底部状态

⸻

2.4 联系人搜索

在会话列表或聊天窗口中输入 /search 进入联系人搜索视图。

示意：

Search Contacts
Search: li
> 李雷
  李老师
  Liam
  项目A群
online │ search: li │ 4 results │ ↑/↓ select │ Enter open │ Esc back

支持操作：

输入文字     更新搜索关键词
↑ / ↓      选择搜索结果
Enter      打开联系人或群聊
Esc        返回上一个视图

3. 需求范围

3.1 MVP 范围

第一阶段需要实现：

* 微信扫码登录
* 登录成功后进入最近会话列表
* 最近会话列表展示若干个会话
* 方向键选择会话
* Enter 打开会话
* 聊天窗口只展示当前会话消息
* 聊天窗口支持直接输入文本发送
* Esc 返回会话列表
* 其他会话新消息只更新未读状态
* 底部状态栏展示当前状态、未读数量、常用操作
* 本地保存联系人、会话、消息
* 支持联系人搜索
* 搜索结果支持方向键选择并打开会话

⸻

3.2 非 MVP 范围

第一阶段不做：

* 全屏复杂 TUI
* 多窗口分栏
* 朋友圈
* 红包
* 转账
* 小程序卡片完整展示
* 公众号完整管理
* 多账号
* 消息漫游
* 拉取微信服务器完整历史记录
* 自动回复机器人
* 群管理
* 富文本编辑
* 完整附件预览

⸻

4. 视图设计

系统包含四个主要视图：

Login View
Conversation List View
Chat View
Contact Search View

4.1 Login View

职责：

* 展示二维码
* 展示登录状态
* 展示扫码确认状态
* 登录成功后切换到会话列表

⸻

4.2 Conversation List View

职责：

* 展示最近会话
* 展示未读状态
* 支持方向键选择
* 支持 Enter 打开会话
* 支持进入搜索
* 接收新消息后更新列表状态

⸻

4.3 Chat View

职责：

* 展示当前会话消息
* 支持文本输入和发送
* 当前会话新消息直接展示
* 非当前会话新消息只更新状态栏
* 支持 Esc 返回会话列表

⸻

4.4 Contact Search View

职责：

* 输入关键词搜索联系人和群聊
* 展示搜索结果
* 支持方向键选择
* 支持 Enter 打开会话
* 支持 Esc 返回上一视图

⸻

5. 输入规则

不同视图下，键盘输入含义不同。

会话列表

↑ / ↓      移动选择
Enter      打开选中会话
/search    进入搜索
q          退出

聊天窗口

普通文本   聊天输入
Enter      发送消息
Esc        返回会话列表
/search    进入搜索

搜索视图

普通文本   搜索关键词
Backspace  删除关键词
↑ / ↓      移动选择
Enter      打开选中联系人
Esc        返回上一视图

⸻

6. 消息处理规则

核心规则：

消息事件不直接输出到终端。
所有消息先进入 Store，再更新 State，最后由 Renderer 决定如何展示。

收到新消息后：

1. 解析消息
2. 保存到本地消息库
3. 更新对应会话的 lastMessage / unreadCount
4. 判断当前视图
5. 触发屏幕重绘

展示规则：

当前在会话列表：
  更新会话列表
当前在聊天窗口，且消息属于当前会话：
  展示到当前聊天窗口
当前在聊天窗口，但消息属于其他会话：
  不展示正文，只更新底部未读状态
当前在搜索窗口：
  不打断搜索，只更新未读状态

⸻

7. 状态栏需求

底部状态栏需要在每个视图中展示当前状态。

会话列表：

online │ unread 4 │ ↑/↓ select │ Enter open │ /search │ q quit

聊天窗口：

online │ 当前: 老板 │ unread: 项目A群(3), 妈妈(1) │ Esc chats

搜索窗口：

online │ search: li │ 4 results │ ↑/↓ select │ Enter open │ Esc back

状态栏需要体现：

* 在线状态
* 当前视图
* 当前会话
* 总未读数
* 其他会话未读摘要
* 当前可用操作提示

⸻

8. 大致架构

建议采用以下分层：

Protocol Layer
  ↓
Adapter Layer
  ↓
Store Layer
  ↓
State Layer
  ↓
Input Layer
  ↓
Renderer Layer

⸻

8.1 Protocol Layer

负责对接微信 Web 协议。

职责：

* 扫码登录
* 保持连接
* 接收消息
* 发送消息
* 获取联系人
* 获取群聊
* 处理登出和异常

⸻

8.2 Adapter Layer

负责屏蔽底层协议差异。

职责：

* 统一联系人模型
* 统一会话模型
* 统一消息模型
* 处理私聊、群聊、自己发送消息的识别
* 处理底层 ID 不稳定问题

⸻

8.3 Store Layer

负责本地持久化。

存储内容：

* 登录会话数据
* 联系人
* 群聊
* 会话
* 消息
* 附件元数据

说明：

* 本地历史只来自程序接收和保存过的消息
* 不承诺拉取微信服务器完整历史记录

⸻

8.4 State Layer

负责维护当前 UI 状态。

核心状态：

currentView
previousView
activeConversationId
conversationListSelectedIndex
searchKeyword
searchSelectedIndex
unreadMap
connectionState

⸻

8.5 Input Layer

负责处理键盘输入。

职责：

* 根据当前视图解释按键
* 处理方向键
* 处理 Enter
* 处理 Esc
* 处理聊天输入
* 处理搜索输入
* 处理命令输入

核心原则：

同一个按键在不同视图中可以有不同语义。

⸻

8.6 Renderer Layer

负责所有终端输出。

职责：

* 清屏
* 渲染登录页
* 渲染会话列表
* 渲染聊天窗口
* 渲染搜索窗口
* 渲染底部状态栏
* 渲染输入行

强约束：

只有 Renderer 可以写 stdout。
其他模块禁止直接 console.log。

⸻

9. 实现目标

2.0 版本完成后，应达到以下效果：

1. 用户扫码登录后看到最近会话列表
2. 用户可以用方向键选择会话
3. 用户按 Enter 进入会话
4. 屏幕只展示当前会话内容
5. 用户可以直接输入文字聊天
6. 当前会话的新消息会显示在聊天窗口
7. 其他会话的新消息不会污染当前屏幕
8. 底部状态栏能提示其他会话的新消息
9. 用户按 Esc 返回会话列表
10. 用户可以通过搜索找到联系人并进入会话

⸻

10. 核心原则

这版的关键不是增加更多功能，而是建立正确的终端交互模型：

当前屏幕只展示当前任务；
消息不直接打印；
所有输出由状态驱动；
会话切换通过清屏重绘完成；
方向键只在列表和搜索视图中选择；
聊天窗口专注于当前会话输入和阅读。

一句话总结：

2.0 版本要把微信 CLI 从“消息流日志”升级为“轻量会话工作台”。
